# 前端界面功能梳理

## 当前界面结构

- 顶部命令栏：展示平台名称、当前项目、三步流程状态，并提供导入系统模型、新建空白模型、注入故障、重置画布、保存模型、保存结果等主操作。
- 左侧组件库：按系统建模、故障注入、测量仪器、连接线四组组织可拖拽组件，支持组件拖入画布和切换连接线类型。
- 中心建模画布：提供画布页签、选择/平移/缩放/整理/模型检查工具、面包屑、节点画布、连线层、子系统封装、当前连接模式和缩放控制。
- 仿真控制区：设置数据集名称、仿真时长、仿真步长、实时模式，并提供初始化、运行、步进、暂停、终止等任务控制。
- 右侧属性面板：按属性、参数配置、故障设置、输出配置组织当前选中组件或连线的配置项。
- 底部状态区：集中展示日志、告警、仿真结果、性能趋势，以及组件数量、故障模型、仿真状态和运行时间。
- 弹窗工作流：包含故障库注入、故障模板建模、电气层故障注入、CAN 协议层故障注入、参数配置、示波器和 Python 文件绑定。

## 选用设计文档

选用 `docs/design-md/ibm/DESIGN.md`，对应 IBM / Carbon Design System 风格。

选择原因：

- 亮色白底、灰阶分层和 IBM Blue 适合工程软件的高密度信息界面。
- 扁平、低圆角、少阴影的界面语言更适合建模平台，能够减少视觉噪声。
- 单一主色加语义色的规则能强化“建模、注入、仿真分析”的流程优先级。
- Carbon 的表格、面板、按钮和输入控件模式适合日志、属性配置、参数输入等工作台场景。

## 改造原则

- 保留现有 DOM id、运行时事件和 Vue 组件边界，不重写仿真、画布、故障注入逻辑。
- 新增最终样式覆盖层 `src/styles/ibm-workbench.css`，便于回退与对照。
- 主色使用 IBM Blue `#0f62fe`，背景采用白色和浅灰，语义色仅用于成功、告警、故障和运行状态。
- 控件采用 0 到 4px 小圆角、细边框、明确焦点态，弱化大面积阴影和渐变。
- 保持工程工作台的信息密度，优先提升可扫读性、操作分组和状态识别。
